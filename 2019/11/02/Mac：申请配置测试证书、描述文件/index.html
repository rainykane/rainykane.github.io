Mac：申请配置测试证书、描述文件

十一月 02, 2019
  • Mac OS

背景

第一次使用Cordova在Mac上打包H5项目时，遇到了问题，此前没有在Mac做过任何开发。在直接用cordova打ipa测试包的时候就报错了，“require a development team”。并有了后续这些操作，在此做记录备忘。

步骤

  1. 准备一个苹果的开发者账号
  2. 生成CRS（Certificate Signing Request）文件
  3. 申请开发测试证书
  4. 申请描述文件
  5. 在Xcode中为项目配置描述文件

操作流程

一、准备一个苹果的开发者账号

苹果开发者中心： developer.apple.com

需要注意的是，注册开发者账号目前的最低价格是99🔪每年。而没有开发者账号的话是无法通过正常的操作来打出ipa包的，下面的流程也就无法进行。

注册开发者账号：使用Apple ID登录苹果开发者中心，在“Account”页面下有一个“Join the Apple Developer Program”的区域，然后按流程走，注册付完费就可以正常使用了。

二、生成CRS（Certificate Signing Request）文件

后续的申请证书时需要上传该文件。

在Mac中打开“钥匙串访问” -> “证书助理” -> “从证书颁发机构请求证书…”

在接下来的弹出框中勾选，“存储到磁盘”，“CA电子邮件地址”则变成“非必需”了

三、申请开发测试证书

此步骤的意义类似Github在配置SSH key。

  1. 在苹果开发者中心中找到”Account” -> “Certificates,Identifiers & Profiles”页，点击”+”号，添加Certificates
  2. 选择”IOS App Development”(根据需要选择)，下一步
  3. 上传已经生成的CSR文件
  4. 下载到mac，双击安装即可。

四、申请描述文件（Provisioning Profile）

描述文件中，包含几个信息：App ID、设备 ID 和 证书

于是在之前的基础上只需要在申请Profiles之前先在苹果开发者中心中创建App ID 和 设备，过程相当简单

App ID : “Account” -> “Certificates,Identifiers & Profiles” -> “Identifiers”

设备 ： “Account” -> “Certificates,Identifiers & Profiles” -> “Devices”

开始申请Profile

  1. 在苹果开发者中心中找到”Account” -> “Certificates,Identifiers & Profiles” -> “Profiles”
  2. 通过”+”号新建，依次选择开发场景，AppID，证书，设备，并填写文件名字
  3. 下载

五、在Xcode中为项目配置描述文件

  1. 在Xcode中的项目配置，”Signing & Capabilities” -> 取消勾选”Automatically manage signing”选项
  2. 确保”Bundle Identifier”中的app id 和创建描述文件时所选择的一致
  3. 选择”Provisioning Profile” -> “Import Profile…”
