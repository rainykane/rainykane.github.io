框架特性

hexo主题选择很多，质量大多数也很高
基于Markdown
解析生成速度快
省去搭建后台的麻烦
域名不需要备案

搭建流程

1. 买域名

  • 选购域名
  • 实名认证

2. 创建github pages

  • 直接在github上创建一个{username}.github.io的仓库（public并且带个Readme）
  • 然后在仓库的settings下拉找到GitHub Pages的配置，选择一个主题
  • 通过{username}.github.io就可以访问到啦

3. 搭建Hexo环境并输出博客静态页

  • 使用 npm install -g hexo 全局安装hexo
  • 在合适的位置使用hexo init初始化一个hexo项目并cd进去
  • 找到合适的Hexo主题并使用
  • 在_config.yml中配置github的地址以便远程提交
  • 通过hexo g | hexo s | hexo d命令，实现静态页面输出 -> 预览 -> 上传到github

4. 绑定域名

  • 通过ping xxx.github.io来获取github pages的ip
  • 解析域名：选择cname，并在示例值填上ip
  • 绑定域名：在settings的github pages的配置中填上域名

具体Hexo的使用参考Hexo官方文档
